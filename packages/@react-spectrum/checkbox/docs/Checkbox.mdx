## Content

Checkboxes can have a visual label, provided by passing a renderable child to the Checkbox.

### Accessibility

In certain cases a visible label isn't needed. For example, a checkbox within a cell to select a row within a table should
use the aria-label prop to explicitly set the accessibility name for the checkbox,
or use aria-labelledby to provide a space-delimited list of id references for elements that label checkbox.

### Internationalization

To internationalize a Checkbox, a localized string should be set as the child content of the Checkbox.
For languages that are read right to left (e.g. Hebrew and Arabic), the Checkbox is placed on the right side of the text.

## Value

A Checkbox's state is unchecked by default. An uncontrolled checked state can be provided by using the `defaultSelected` prop.
Alternatively, providing an `isSelected` prop creates a Checkbox with a controlled checked state.

```tsx example
<Checkbox defaultSelected> Uncontrolled </Checkbox>
```

```tsx example
<Checkbox isSelected> Controlled </Checkbox>
```

### isIndeterminate
[View guidelines](https://spectrum.adobe.com/page/checkbox/#Mixed-value)

When a Checkbox is indeterminate, it overrides the selection state.
The indeterminate visual state will be maintained regardless of click events.

```tsx example
<Checkbox isIndeterminate> Checkbox Label </Checkbox>
```

## Events

Checkboxes accept a user defined `onChange` prop, triggered whenever the Checkbox is clicked.
The example below uses `onChange` to alert the user to changes in the checkbox's state.

```jsx example
function Example() {
  let [selected, setSelection] = React.useState('false');

  let toggle = (value) => {
    value ? setSelection('true') : setSelection('false');
  }

  return (
    <div>
    <div> The Checkbox is checked: {selected} </div>
    <Checkbox
      onChange={toggle}
      aria-label="This checkbox has no visible label"
    />
    </div>
  );
 }
```

## Validation

Checkboxes can display a validation state to communicate to the user whether the current value is valid or invalid.
Implement your own validation logic in your app and pass either `"valid"` or `"invalid"` to the Checkbox via the `validationState` prop.

```tsx example
<Checkbox validationState="invalid"> Checkbox Label </Checkbox>
```

```tsx example
<Checkbox validationState="valid"> Checkbox Label </Checkbox>
```
