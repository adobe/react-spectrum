# useRadioGroup

<PageDescription>{docs.exports.useRadioGroup.description}</PageDescription>

This example uses `useRadioGroup` for the group and `useRadio` for each option. The group state is shared with each radio through React context.

```tsx render
"use client";
import React from 'react';
import {useRadioGroup, useRadio} from 'react-aria/useRadioGroup';
import {useFocusRing} from 'react-aria/useFocusRing';
import {mergeProps} from 'react-aria/mergeProps';
import {useRadioGroupState} from 'react-stately/useRadioGroupState';
import 'vanilla-starter/theme.css';

let RadioContext = React.createContext<any>(null);

function RadioGroup(props) {
  let {children, label} = props;
  let state = useRadioGroupState(props);
  let {radioGroupProps, labelProps} = useRadioGroup(props, state);

  return (
    <div {...radioGroupProps} style={{color: 'var(--text-color)'}}>
      <span {...labelProps}>{label}</span>
      <RadioContext.Provider value={state}>
        {children}
      </RadioContext.Provider>
    </div>
  );
}

function Radio(props) {
  let state = React.useContext(RadioContext);
  let ref = React.useRef(null);
  let {inputProps} = useRadio(props, state, ref);
  let {isFocusVisible, focusProps} = useFocusRing();

  return (
    <label style={{display: 'flex', alignItems: 'center', gap: 8}}>
      <span
        style={{
          display: 'inline-flex',
          borderRadius: '50%',
          outline: isFocusVisible ? '2px solid var(--focus-ring-color)' : 'none',
          outlineOffset: 2
        }}>
        <input {...mergeProps(inputProps, focusProps)} ref={ref} style={{margin: 0, outline: 'none'}} />
      </span>
      {props.children}
    </label>
  );
}

<RadioGroup label="Favorite pet">
  <Radio value="dogs">Dogs</Radio>
  <Radio value="cats">Cats</Radio>
</RadioGroup>
```

## Features

Radio groups can be built with the [&lt;input&gt;](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/input) HTML element and the `radio` type, but this can be difficult to style. `useRadioGroup` and `useRadio` help achieve accessible radio groups that can be styled as needed.

* Radio groups are exposed to assistive technology via ARIA, built with native HTML `<input>` elements
* Full support for browser features like form autofill
* Keyboard focus management and cross browser normalization, including arrow key navigation between radios
* Labeling support for accessibility, and support for description and error message help text
* Support for native HTML constraint validation with customizable UI, custom validation functions, realtime validation, and server-side validation errors

## State management

`useRadioGroup` requires a `state` object created with the `useRadioGroupState` hook from `react-stately`, which tracks the selected value. The state is passed to each `useRadio`, typically via React context.
