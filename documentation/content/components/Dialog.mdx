## Overview

An **Dialog** in Spectrum is modal window that forces the user to take some action. They can have up to three buttons for the user
to choose from.

## Example

Dialogs are typically used with a `ModalTrigger` to open them when clicking on an action, e.g. a button or menu item.

<ModalTrigger>
  <Button label="Open Dialog" variant="primary" />
  <Dialog title="Dialog title" confirmLabel="OK">
    Content of the dialog
  </Dialog>
</ModalTrigger>

```jsx
<ModalTrigger>
  <Button label="Open Dialog" variant="primary" />
  <Dialog title="Dialog title" confirmLabel="OK">
    Content of the dialog
  </Dialog>
</ModalTrigger>
```

## Buttons

Dialogs can have up to three buttons: 2 action buttons, and a cancel button. All of them are optional. When one of the confirm buttons
is clicked, `onConfirm` is called with the button that was clicked: either "primary" or "secondary". When the cancel button is clicked,
`onCancel` is called. You can also optionally disable the confirm button using the `confirmDisabled` prop.

Enabling the `keyboardConfirm` prop makes pressing the Enter key execute the default action for the Dialog.
It is the equivalent of pressing the “Confirm” button.

<ModalTrigger>
  <Button label="Open Dialog" variant="primary" />
  <Dialog
    title="Dialog title"
    confirmLabel="Keep Both"
    secondaryLabel="Replace"
    cancelLabel="Cancel"
    keyboardConfirm
    onConfirm={action => console.log(action)}
    onCancel={() => console.log('cancel')}>
    Content of the dialog
  </Dialog>
</ModalTrigger>

```jsx
<ModalTrigger>
  <Button label="Open Dialog" variant="primary" />
  <Dialog
    title="Dialog title"
    confirmLabel="Keep Both"
    secondaryLabel="Replace"
    cancelLabel="Cancel"
    keyboardConfirm
    onConfirm={action => console.log(action)}
    onCancel={() => console.log('cancel')}>
    Content of the dialog
  </Dialog>
</ModalTrigger>
```

## Variants

There are many Dialog variants to choose from in Spectrum. The `variant` prop controls the main variant of the Dialog.

### Confirmation

This is the default variant for dialogs. Confirmation dialogs should be used when asking users to confirm a choice.
The confirmation dialog has a blue button (call to action button) to highlight the strong preference for buttons.

<ModalTrigger>
  <Button label="Open Dialog" variant="primary" />
  <Dialog
    variant="confirmation"
    title="Dialog title"
    confirmLabel="OK"
    cancelLabel="Cancel"
    confirmDisabled>
    Content of the dialog
  </Dialog>
</ModalTrigger>

```jsx
<ModalTrigger>
  <Button label="Open Dialog" variant="primary" />
  <Dialog
    variant="confirmation"
    title="Dialog title"
    confirmLabel="OK"
    cancelLabel="Cancel">
    Content of the dialog
  </Dialog>
</ModalTrigger>
```

## Accessibility

Dialogs automatically hide content behind them from screen readers while open, block scrolling of the page, and focus the modal.
Dialogs automatically close on pressing the <kbd>Escape</kbd> key, and when the `keyboardConfirm` prop is enabled, the default action is
automatically triggered when pressing the <kbd>Enter</kbd> key. You can also automatically focus either the `cancel` or
`confirm` button when a dialog opens using the `autoFocusButton` prop.
When the a dialog closes, keyboard focus will be restored to the element that had focus when the dialog opened.

## Internationalization

To internationalize a dialog, provide localized strings for the title, buttons, and body of the dialog.
