## Overview

A **ButtonGroup** represents list of action buttons a user can take. Buttons inside the group can be clicked or tapped to perform an action. ButtonGroup in Spectrum have several props for different uses and multiple levels of loudness for various attention-getting needs.
ButtonGroup allows two kinds of Button variants, tool and action. If a button is not one of these, its variant will be overridden to be action.

## Example

<ButtonGroup>
  <Button label="Brush" value="Brush" icon={<Brush />} />
  <Button label="Bell" value="Bell" icon={<Bell />} />
  <Button label="AddCircle" value="AddCircle" icon={<AddCircle />} />
</ButtonGroup>

```jsx
<ButtonGroup>
  <Button label="Brush" value="Brush" icon={<Brush />} />
  <Button label="Bell" value="Bell" icon={<Bell />} />
  <Button label="AddCircle" value="Add" icon={<AddCircle />} />
</ButtonGroup>
```

## Group Orientation

Spectrum ButtonGroup can displayed vertically, Default is horizontal. 'orientation' prop is passed with appropriate orientation value. This can also be used with different variant like `tool`.

<ButtonGroup orientation="vertical">
  <Button label="Brush" value="Brush" icon={<Brush />} />
  <Button label="Bell" value="Bell" icon={<Bell />} />
  <Button label="AddCircle" value="AddCircle" icon={<AddCircle />} />
</ButtonGroup>

```jsx
<ButtonGroup orientation="vertical">

  <Button label="Bell" value="Bell" icon={<Bell />} />
  <Button label="AddCircle" value="Add" icon={<AddCircle />} />

```

## Multiple Selection

To allow multiple selection of Buttons in ButtonGroup 'multiple' props is passed. Allows selection of more than more button at a time.

<ButtonGroup multiple>
  <Button label="Brush" value="Brush" icon={<Brush />} />
  <Button label="Bell" value="Bell" icon={<Bell />} />
  <Button label="AddCircle" value="AddCircle" icon={<AddCircle />} />
</ButtonGroup>

```jsx
<ButtonGroup multiple>

  <Button label="Bell" value="Bell" icon={<Bell />} />
  <Button label="AddCircle" value="Add" icon={<AddCircle />} />

```

## State

### Disabled

All the ButtonGroup supports `disabled` prop like other props. It will disable all the Buttons in the group and the changes the visual representation as well.

<ButtonGroup disabled>
  <Button label="Brush" value="Brush" icon={<Brush />} />
  <Button label="Bell" value="Bell" icon={<Bell />} />
  <Button label="AddCircle" value="AddCircle" icon={<AddCircle />} />
</ButtonGroup>


### Readonly
The Readonly prop disables selections of any kind, instead of onChange being fired, all clicks go through the onClick handler.

<ButtonGroup readOnly>
  <Button label="Brush" value="Brush" icon={<Brush />} />
  <Button label="Bell" value="Bell" icon={<Bell />} />
  <Button label="AddCircle" value="AddCircle" icon={<AddCircle />} />
</ButtonGroup>

```jsx
<ButtonGroup readOnly>
  <Button label="Brush" value="Brush" icon={<Brush />} />
  <Button label="Bell" value="Bell" icon={<Bell />} />
  <Button label="AddCircle" value="Add" icon={<AddCircle />} />
</ButtonGroup>
```

## Accessibility

### WAI-ARIA Design Pattern

The ButtonGroup accessibility implementation depends on selection mode.

#### Default (mutually exclusive selection)

By default, Button selection is mutually exclusive, and ButtonGroup implements the [RadioGroup design pattern per WAI-ARIA 1.1](https://www.w3.org/TR/wai-aria-practices/#radiobutton); the ButtonGroup `role` for assistive technology is `radiogroup`, and Button elements use the `radio` `role` and `aria-checked` to indicate the toggled state.

The ButtonGroup should be labelled using either `aria-label` or `aria-labelledby`.

<ButtonGroup aria-label="Text Alignment">
  <Button
    value="TextAlignLeft"
    aria-label="Left"
    icon={<TextAlignLeft />} />
  <Button
    value="TextAlignCenter"
    aria-label="Center"
    icon={<TextAlignCenter />} />
  <Button
    value="TextAlignRight"
    aria-label="Right"
    icon={<TextAlignRight />} />
  <Button
    value="TextAlignJustify"
    aria-label="Justify"
    icon={<TextAlignJustify />} />
</ButtonGroup>

```jsx
<ButtonGroup aria-label="Text Alignment">
  <Button
    value="TextAlignLeft"
    aria-label="Left"
    icon={<TextAlignLeft />} />
  <Button
    value="TextAlignCenter"
    aria-label="Center"
    icon={<TextAlignCenter />} />
  <Button
    value="TextAlignRight"
    aria-label="Right"
    icon={<TextAlignRight />} />
  <Button
    value="TextAlignJustify"
    aria-label="Justify"
    icon={<TextAlignJustify />} />
</ButtonGroup>
```

#### Multiple Selection

When multiple selection is set using the `multiple` prop, the ButtonGroup `role` for assistive technology is `group`, and Button elements use the `checkbox` `role` and `aria-checked` to indicate the toggled state.

The ButtonGroup should be labelled using either `aria-label` or `aria-labelledby`.

<ButtonGroup aria-label="Text Styling" multiple>
  <Button
    value="Italic"
    aria-label="Italic"
    icon={<TextItalic />} />
  <Button
    value="Underline"
    aria-label="Underline"
    icon={<TextUnderline />} />
  <Button
    value="Strikethrough"
    aria-label="Strikethrough"
    icon={<TextStrikethrough />} />
</ButtonGroup>

```jsx
<ButtonGroup aria-label="Text Styling" multiple>
  <Button
    value="Italic"
    aria-label="Italic"
    icon={<TextItalic />} />
  <Button
    value="Underline"
    aria-label="Underline"
    icon={<TextUnderline />} />
  <Button
    value="Strikethrough"
    aria-label="Strikethrough"
    icon={<TextStrikethrough />} />
</ButtonGroup>
```

#### No Selection

With no selection using the `readOnly` prop, ButtonGroup implements the [Toolbar design pattern per WAI-ARIA 1.1](https://www.w3.org/TR/wai-aria-practices/#toolbar); the ButtonGroup `role` for assistive technology is `toolbar`, and Button elements retain their implicit `button` `role`.

The ButtonGroup should be labelled using either `aria-label` or `aria-labelledby`.

<ButtonGroup aria-label="Tools" readOnly>
  <Button label="Brush" value="Brush" icon={<Brush />} />
  <Button label="Bell" value="Bell" icon={<Bell />} />
  <Button label="AddCircle" value="AddCircle" icon={<AddCircle />} />
</ButtonGroup>

```jsx
<ButtonGroup aria-label="Tools" readOnly>
  <Button label="Brush" value="Brush" icon={<Brush />} />
  <Button label="Bell" value="Bell" icon={<Bell />} />
  <Button label="AddCircle" value="Add" icon={<AddCircle />} />
</ButtonGroup>
```

### manageTabIndex

By default, ButtonGroup uses a roving tab index so that only one button in the group is included in the tab navigation order at a time.
This behavior can be overridden, so that all buttons will be included in the tab navigation order, by setting the `manageTabIndex` prop to `false`.

<ButtonGroup aria-label="Tools" readOnly manageTabIndex={false}>
  <Button label="Brush" value="Brush" icon={<Brush />} />
  <Button label="Bell" value="Bell" icon={<Bell />} />
  <Button label="AddCircle" value="AddCircle" icon={<AddCircle />} />
</ButtonGroup>

```jsx
<ButtonGroup aria-label="Tools" readOnly manageTabIndex={false}>
  <Button label="Brush" value="Brush" icon={<Brush />} />
  <Button label="Bell" value="Bell" icon={<Bell />} />
  <Button label="AddCircle" value="Add" icon={<AddCircle />} />
</ButtonGroup>
```
